Remarks
Firstly, I'd like to apologize for the lack of direct documentation. Due to the sheer number of permutations possible between ranges and type constraints, this header makes use of internal macros to generate all the concepts. Because of this, doxygen can't generate the documentation for the individual concepts. Ideally, this would be better expressed as a set of concept templates; however, the C++ standard does not allow for this as concepts cannot be template arguments. There are work-arounds, but their semmantics are just as if not more noisy than the normal boiler-plate, and the error messages are often indecipherable or lack critical information, such as why the concept was not satisfied. Please bear with with me for this explanation. Once you understand the syntax, they are quite intuitive to use.

#include <fgl/types/range_constraints.hpp> defines concepts that allow for constraining ranges based on the value or reference type without the template<> requires boiler-plate noise.

Range constraints are conjunctions between a range concept and a type concept which constrains either the std::ranges::range_value_t or std::ranges::range_reference_t, depending on which is most applicable to the concept.

Generated concept syntax:

  • A conjunction between a range and type concept

    template <T_range> concept fgl::RANGE CONCEPT NAME_TYPE CONCEPT NAME

    Template Parameters
    T_rangethe range being tested
    Parameters
    RANGE CONCEPT NAMEthe name of the concept which constrains the range, such as contiguous_range
    TYPE CONCEPT NAMEthe name of the concept which constrains the type, such as integral

    So if you wanted to test if a range was a contiguous range of integrals, you would write fgl::contiguous_range_integral<std::vector<int>>.

  • A conjunction between a range and a type concept which requires a comparison to another type, such as std::same_as

    template <T_range, T_compare> concept fgl::RANGE CONCEPT NAME_TYPE CONCEPT NAME

    Template Parameters
    T_rangethe range being tested
    T_comparethe type being compared to
    Parameters
    RANGE CONCEPT NAMEthe name of the concept which constrains the range, such as forward_range
    TYPE CONCEPT NAMEthe name of the concept which constrains the type, such as same_as

    So if you wanted to test if a range was a forward range of ints, you would write fgl::forward_range_same_as<std::list<int>, int>.

  • A conjunction between a range and a type concept which requires a comparison to the underlying type of another range, such as std::same_as<std::ranges::range_value_t<T_compare_range>> or std::same_as<std::ranges::range_reference_t<T_compare_range>>

    template <T_range, T_compare_range> concept fgl::RANGE CONCEPT NAME_ TYPE CONCEPT NAME_range

    Template Parameters
    T_rangethe range being tested
    T_compare_rangethe range whose type is being compared to
    Parameters
    RANGE CONCEPT NAMEthe name of the concept which constrains the range, such as bidirectional_range
    TYPE CONCEPT NAMEthe name of the concept which constrains the type, such as same_as
    So if you wanted to test if a range was a bi-directional range whose underlying type was the same as the type of another range, you would write fgl::bidirectional_range_same_as_range<std::array<int, 3>, std::vector<int>>.

Supported range concepts:

  • std::ranges::range
  • std::ranges::borrowed_range
  • std::ranges::sized_range
  • std::ranges::bidirectional_range
  • std::ranges::random_access_range
  • std::ranges::contiguous_range
  • std::ranges::common_range
  • std::ranges::viewable_range

Supported type concepts:

  • std::swappable_with<T&>
  • std::assignable_from<T>
  • std::same_as<T>
  • std::derived_from<T>
  • std::convertible_to<T>
  • std::common_with<T>
  • std::integral
  • std::signed_integral
  • std::unsigned_integral
  • std::floating_point
  • fgl::traits::not_same_as<T>
  • fgl::traits::numeric_type
  • fgl::traits::byte_type
  • fgl::traits::pointer_to_byte
  • fgl::traits::pointer_to_non_void

Refer to libFGL's Type Traits for the fgl::traits concepts

Type concepts and the _range suffix:

Each comparison concept also has a variant with the _range suffix, whose template arguments are <T_range, T_compare_range>. This variant will retrieve the constraint comparison type from T_compare_range's value or reference type; which ever is used to retrieve the type being constrained from T_range.
