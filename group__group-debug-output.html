Detailed Description

libFGL debug output fascility

The <fgl/debug/output.hpp> header is for the configuration and access of the libFGL debug output stream. It establishes a configurable, type-specific output system and surrounding infrastructure. The output system is designed to be flexible and easy to use, though it can appear a bit daunting without a high-level overview.

While intended as a "debug output system", the libFGL output system could be used for any purpose; including logging. For this reason, several output thresholds are provided in the form of fgl::debug::priority. The output stream defaults to std::cout but can be redirected to any std::ostream.

Output is sent on an fgl::debug::output_channel which is a type that is used to identify and configure the channel. Each channel has a priority, enabled state, and name.

The fgl::debug::output class and its corresponding instance of the same name is used to configure the output system, as well as conditionally access the output stream. Access will only be granted if the channel can_send(), at which point the type being sent to the output stream is formatted using a corresponding fgl::debug::output_formatter; the returned value of which is sent to the output stream. A type which is both a channel and a formatter is called an fgl::debug::output_handler.

The fgl::debug::output_config class template is a generic output handler. fgl::debug::output::operator()(), the most easy-to-use way of sending to the debug output stream, relies on the existance of an output_config<T> output handler. For this reason, and by convention, the output_config template is often specialized as an output handler for a specific type; often centralizing configuration options such as channel name, priority, formatters, etc.

See also
The example programs example/fgl/debug/output_simple.cpp and example/fgl/debug/output_advanced.cpp

static bool fgl::debug::output::can_send ( )
inlinestaticnoexcept

Checks whether or not a channel is permitted to send output.

Whether a channel may send output is determined by three factors:

  • whether or not fgl::debug::output::enabled is true
  • whether or not the channel's enabled() state is true
  • and whether or not the channel's priority_level() is greater than or equal to the fgl::debug::output::priority_threshold

◆ stream

output_stream_t fgl::debug::output::stream
inlinestatic

The debug output stream (redirect and flush only)

The output stream to which all libFGL debug output is sent. To prevent misuse of the output stream, this member prevents direct access. The stream may only be accessed thru other fgl::debug::output methods which require an output_channel in order to send output.

Note
The default stream is std::cout
Examples
example/fgl/debug/output_advanced.cpp, example/fgl/debug/output_simple.cpp, and example/fgl/debug/stopwatch.cpp.
