Stopwatch Recording Methods

These methods are used to start, lap, stop, and reset the stopwatch.

Each pairing of start() and stop() calls will record the elapsed time between their time points as a lap duration. The lap() method is used to record a lap duration without stopping the stopwatch; its time point argument becomes the new start time.

Note
While these methods are constexpr, the default argument clock_t::now() may not be depending on the clock type. At the time of writing (C++20) there are no standard constexpr clocks.

Stopwatches have an internal state to prevent undefined behavior, ensure correctness by enforcing a conceptually valid order of operations. Calling the methods start(), lap(), stop(), and stop_without_record() will assert that the stopwatch is in a valid state for that method's intended function. For example: the stopwatch cannot be stopped until after it has been started, and once it has been started it cannot be started again until t's been either stopped or reset. This is an artificial limitation which exists soley to prevent the accidental misuse which may lead to undefined behavior or invalid time records.

Note
To ensure optimal performance, the internal state mutations and assertions will be disabled and optimized out when NDEBUG is defined.
Warning
When the assertions are disabled by defining NDEBUG, it becomes possible to invoke undefined behavior if a method is called when the stopwatch is in a state which is invalid for that method! For example, calling lap() or stop() when the stopwatch is already stopped, or calling start() when it has already been started.

Detailed Description

A model stopwatch and output formatting fascilities.

The <fgl/debug/stopwatch.hpp> header provides a template for a generic stopwatch capable of recording elapsed time, a convenience alias for a std::chrono::steady_clock stopwatch, and relevant formatting fascilities which support sending output to libFGL's Debug Output System.

See also
The example program example/fgl/debug/stopwatch.cpp

inlineconstexpr

Without stopping the watch, records a lap whose duration is time_point subtracted from the start time, and then updates the start time.

Parameters
time_pointThe time point which will be subtracted from the start time to produce a lap duration. This time point will become the new start time after the lap duration has been recorded.
Note
The stopwatch must be "ticking". Lapping a stopwatch is only valid after it has been started.
Remarks
For high precision timing, it's recommended to stop() and then start() the stopwatch rather than calling lap() as the time between emplacing the lap duration and updating the start time may be significant, especially if the number of laps grows beyond what was reserved when the stopwatch was constructed.
Examples
example/fgl/debug/stopwatch.cpp.

◆ elapsed()

template<fgl::traits::steady_clock T_clock = std::chrono::steady_clock>
constexpr duration_t fgl::debug::generic_stopwatch< T_clock >::elapsed ( ) const
inlineconstexpr
Returns
The sum of all recorded lap durations.

Equivalent to calling elapsed_between_laps(0, number_of_laps()).

Note
The stopwatch must be in either a "ticking" or "stopped" state.
The elapsed time is **NOT** the last stop time point subtracted from the first start time point. It is the sum of all recorded laps. There may be time between start()s and stop()s which isn't represented; this is an intentional design choice.
