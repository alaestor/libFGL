Detailed Description

template<typename T>
class fgl::debug::output_config< T >

Generic output handler template.

The generic definition of this template satisfies the output_handler concept and can be used to configure the generic channels and formatters on a per-type basis.

Note
If a type cannot use the generic output_config, a custom formatter and output channel must be defined. Ideally, they would be combined in a specialization of output_config<T> which satisfies the fgl::debug::output_handler concept so that T may be used with fgl::debug::output(t).

By convention, this template is also specialized as the output configuration point for all libFGL types which interact with the libFGL output system. For example, an fgl::T might define a specialization of output_config<fgl::T> which is can be used to turn on or off the fgl::T related output channel, or set a user-defined custom formatter for fgl::T.

Note

Output from generic type channels can be disabled by setting fgl::debug::disable_generic_output_channels to true.

By default, the channel names are "GENERIC" and the priority is fgl::debug::priority::info.

Attention

For the generic default formatter to function properly, T must satisfy fgl::traits::printable<T_value>. If it doesn't, the default formatter will throw a std::logic_error exception, whose what string will be an attention notice.

Example notice:

ATTENTION! Generated from: static std::string fgl::debug::output_config<T>:: default_formatter(const T&) [with T = my_struct; std::string = std::__cxx11::basic_string<char>]

The generic `output_config` formatter is only suitable for types which satisfy `fgl::traits::printable` by having an ostream `operator<<` overload. If you are seeing this, you should either define such an overload, or define a custom formatter for `T` and assign it to the static `fgl::debug::output_config<T>::format` member variable.
